# Project History And Vision

## Como nacio la idea

La idea nacio a partir de una experiencia real del fundador durante la hospitalizacion de su papa. En esa situacion aparecio una necesidad concreta: cubrir guardias de acompanamiento y cuidado para un paciente hospitalizado durante periodos prolongados, especialmente en turnos nocturnos.

La familia inicialmente absorbio esa carga. Con el paso de los dias, el esfuerzo se volvio pesado y fue necesario contratar apoyo externo. El problema fue que no existia una forma clara, visible y confiable de encontrar ese servicio. La contratacion termino ocurriendo de manera manual, a partir de anuncios impresos en el hospital, llamadas y recomendaciones informales.

Esa experiencia dio visibilidad a un espacio operativo real:

- existe una necesidad recurrente
- existe oferta de cuidadoras y enfermeras
- existe coordinacion humana valiosa
- pero casi todo se mueve por WhatsApp, recomendaciones y memoria operativa

## Lo que se observo del mercado informal

De las conversaciones con una amiga enfermera cercana al ambiente, se identificaron patrones consistentes:

- algunas enfermeras funcionan como administradoras o coordinadoras
- ellas consiguen clientes y distribuyen guardias
- las guardias pueden ser en hospital publico, hospital privado o domicilio
- los turnos suelen ser de 12 horas, pero no siempre
- las tarifas, comisiones y pagos se acuerdan de manera muy manual
- el reporte final no esta estandarizado
- no existe una reputacion sistematizada
- la coordinacion de personal ocurre mucho en grupos de WhatsApp
- una misma persona puede cubrir guardias y coordinar a otras enfermeras

Este comportamiento mostro que el valor inicial no estaba necesariamente en construir un marketplace gigante desde el dia uno, sino en resolver primero la operacion interna de una coordinadora real.

## Vision original amplia

La vision amplia contempla eventualmente un ecosistema mas grande:

- una app o experiencia para familiares que solicitan servicio
- una app o experiencia para enfermeras/cuidadoras
- catalogos y perfiles
- ubicacion, confirmaciones, reportes y reputacion
- gestion de turnos y cobros

Pero se considero poco realista empezar ahi.

## Decisiones estrategicas tempranas

Se tomaron varias decisiones clave para aterrizar el proyecto:

### 1. Empezar por una V1 interna

La primera version no intenta convencer a todo un mercado nuevo. En lugar de eso, busca ayudar a una administradora real a trabajar mejor.

### 2. No perseguir complejidad de plataforma desde el inicio

No se esta construyendo de inicio:

- un marketplace abierto
- una red social de enfermeras
- un sistema clinico completo
- una solucion de pagos
- una app multiusuario compleja

### 3. Favorecer utilidad operativa inmediata

La pregunta guia de la V1 es:

> Esto le facilitaria la vida hoy a una coordinadora de guardias de confianza?

Si la respuesta es no, probablemente no pertenece a esta fase.

## Usuario principal definido

El usuario principal de la V1 es:

- una administradora/coordinadora de guardias de enfermeria

No es el familiar.
No es la flotilla completa de enfermeras.
No es un hospital.

Eso permite decisiones mas claras en UX, alcance, validaciones y lenguaje.

## Nombre e identidad de la V1

Para esta version se cerro esta identidad:

- nombre: `El Cuartel de las Cofias`
- slogan: `Cuidado con corazon`

La identidad de V1 se acepta como interna y cercana. No bloquea que mas adelante el producto evolucione a una marca mas amplia como `Red de Cofias` u otra alternativa.

## Vision de producto a mediano plazo

La vision no es solo “una app para registrar cosas”. La vision es construir una base operativa que despues pueda crecer organica y responsablemente.

Esa evolucion se imagina por etapas:

1. herramienta interna para coordinadora
2. herramienta operativa para un pequeno grupo de enfermeras de confianza
3. experiencia mas estructurada para solicitantes/familiares
4. plataforma mas robusta si el uso real lo justifica

## Limites deliberados

El proyecto toca una zona sensible: salud, cuidado y datos personales. Por eso, desde el principio se acordaron limites fuertes:

- no convertir V1 en expediente clinico
- no capturar mas datos sensibles de los necesarios
- no simular capacidades medicas o regulatorias que no existen
- no disenar funciones por moda si todavia no resuelven operacion real

## Vision de largo plazo

Si el producto demuestra valor real, podria evolucionar a:

- coordinacion de plantillas de confianza
- manejo estandarizado de reportes
- trazabilidad basica de operacion
- soporte a cobros y pagos
- recordatorios y alertas
- crecimiento multiusuario
- integraciones futuras

Pero ese crecimiento debe ocurrir solo cuando el uso real lo pida, no antes.

## Frase guia del proyecto

La filosofia que mejor resume el proyecto hasta ahora es:

> Empezar con algo pequeno, util y real para una coordinadora; crecer solo cuando la operacion lo justifique.
